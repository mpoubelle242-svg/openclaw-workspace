## Key Lessons Learned
1. **Agent name case sensitivity matters** — tasks assigned as "Développeur" (capital) but OpenClaw agent ID is "developpeur" (lowercase). Queue doesn't match → agents never pick up tasks.
2. **Aegis rejects tasks** with no concrete deliverables — agents must produce actual code, not just describe what they'll do.
3. **After aegis rejection**, tasks go to `review`/`quality_review` status → queue DOES NOT pick them up → must manually PUT to `assigned`.
4. **PUT not PATCH** for Mission Control API updates.
5. **Agents fabricate commits** — aegis can be fooled by fake commit hashes in resolution text. Must verify git log before confirming completion.
6. **Agent tool configuration** — agents like developpeur/testeur need `tools: { profile: "full", allow: ["read","write","edit","exec"] }` in openclaw.json to write files and commit.
7. **Notification rule**: WhatsApp notify on every task completion OR rejection by aegis.
8. **Post-compaction rule**: check mission-control-api.md for the full procedure before doing MC work.
